01

Overview

Privacy-first by design

Sentinel AV is a local-first application. The vast majority of its processing — file scanning, heuristics, quarantine, threat logging — happens entirely on your device. Data is only transmitted to external services when you explicitly configure them.

This Privacy Policy describes what information Sentinel AV ("the App", "we", "us") collects and how it is used when you install and run Sentinel AV on your computer. By using the App you agree to this policy.

02

Data We Collect

Sentinel AV collects two categories of data: data stored locally on your device, and data optionally transmitted to third-party services.

Data Type Where Stored Transmitted?
File SHA-256 hashes Local SQLite DB Optional
Threat detections & metadata Local SQLite DB Never
Quarantine file contents Local (AES-256 encrypted) Never
Scan activity logs Local SQLite DB Never
App settings & preferences Local app userData Never
VirusTotal API key OS keychain (keytar) Never
Anthropic API key OS keychain (keytar) Never
File path strings Local only (AI analysis) Optional

03

What Stays Local

The following data never leaves your device under any circumstances:

  • The contents of scanned files
  • Quarantined file data (stored AES-256-CBC encrypted in your local userData folder)
  • Threat detection records, severity scores, and scan history
  • Your API keys — stored exclusively in your operating system's secure keychain via keytar
  • Watch-path configuration and scan schedules
  • All application settings and preferences

Sentinel's two SQLite databases (sentinel.db and signatures.db) reside entirely within your device's userData directory and are never synced or uploaded.

04

External Services

Sentinel AV can optionally communicate with the following third-party services. These integrations are disabled by default and only activate when you provide an API key in Settings.

VirusTotal

When enabled, the SHA-256 hash of a scanned file is sent to VirusTotal's public API for multi-engine threat lookup. File contents are never transmitted — only the hash. VirusTotal's own privacy policy governs data submitted to their service.

Anthropic Claude API (AI Analysis)

When AI analysis is enabled, Sentinel sends heuristic feature data — file metadata, behavioral signals, and file path strings — to the Anthropic Claude API for intelligent threat assessment. Raw file contents are never transmitted. Anthropic's privacy policy governs data submitted to their API.

05

API Key Handling

Your API keys for VirusTotal and Anthropic are stored exclusively using your operating system's secure credential store:

  • Windows: Windows Credential Manager
  • macOS: Keychain
  • Linux: libsecret / GNOME Keyring

Keys are accessed at runtime via keytar and are never written to disk in plaintext, never included in logs, and never transmitted to any server other than the API endpoint they authenticate with.

06

Quarantine & Scanning

When a file is quarantined, Sentinel AV:

  1. Encrypts the file using AES-256-CBC with a key derived via PBKDF2
  2. Moves the encrypted blob to your local userData/quarantine/ directory
  3. Records metadata (original path, threat type, timestamp) in the local database

Quarantined file contents are never transmitted or shared. You may restore or permanently delete quarantined files at any time from the Quarantine panel.

Important: Real-time Watcher

The real-time file watcher monitors file system events in your configured watch paths. It reads file contents only to compute SHA-256 hashes and perform heuristic analysis — all on-device. No content is cached beyond the current scan operation.

07

Data Retention

All data is retained locally until you choose to delete it:

  • Scan history & logs — Stored indefinitely in sentinel.db. You may clear history via Settings.
  • Quarantine files — Retained until you restore or delete them from the Quarantine panel.
  • Signature database — Updated when you run update-signatures; old entries are replaced in-place.
  • API keys — Retained in the OS keychain until you remove them from Settings or uninstall the app.

Uninstalling Sentinel AV removes the application but does not automatically delete data in userData or your OS keychain. To fully remove all data, delete the userData folder and remove credentials from your OS keychain manually.

08

Your Rights

Because Sentinel AV stores data locally on your device, you are in direct control of it at all times:

  • Access: All data is stored in open SQLite databases you can inspect directly.
  • Deletion: Clear history, remove quarantined files, or delete the userData directory at any time.
  • Portability: Copy or export your SQLite databases at any time.
  • Opt-out: Disable any external service integration from Settings at any time without affecting core functionality.

We do not operate servers that hold your personal data, so there is no data-subject request process applicable to Sentinel Security for locally stored information.

09

Sentinel AV is not directed at children under 13. We do not knowingly collect personal information from children. The App does not include features targeting minors.
