Step 1 — Detection Engine

Multi-Layer Threat Detection

Sentinel stacks three independent detection methods to maximize coverage while minimizing false positives.

1
SHA-256 Signature Lookup
Every file is hashed and cross-referenced against the local signature database — a SQLite store fed by MalwareBazaar feeds. Lookups are instant, entirely offline, and require no API key.
Local — no internet required
2
VirusTotal Cross-Check (Optional)
When a VirusTotal API key is configured, the file's hash is submitted for multi-engine consensus. Only the hash is transmitted — file contents never leave your machine. Results are cached to respect the free-tier rate limit.
Hash-only — content stays local
3
Heuristic Scoring (0–100)
The heuristics engine computes a behavioral risk score by examining file structure, entropy, extension mismatches, embedded strings, and file-type characteristics. Scores ≥ 60 trigger AI analysis; scores ≥ 80 auto-quarantine.
Score ≥80 → auto-quarantine
Signature DB Updates
Run npm run update-signatures to pull the latest MalwareBazaar feeds into your local DB.
Entropy Analysis
High-entropy sections within files flag potential packed or encrypted payloads — a common malware evasion technique.
Extension Mismatch Detection
Files claiming to be images or documents but containing executable code are flagged immediately — regardless of their file extension.

Step 2 — Intelligence Layer

Claude AI Threat Analysis

When heuristic scores are borderline, Sentinel consults Claude (claude-opus-4-6) for a structured threat assessment — turning ambiguous signals into actionable verdicts.

1
Heuristic Escalation
Files with a heuristic score between 60–79 are escalated to the AI layer. The engine packages behavioral signals, metadata, and file attributes into a structured prompt.
Trigger: score ≥ 60
2
Claude API Call
Sentinel sends the heuristic feature payload to Anthropic's Claude API. Raw file contents are never included — only derived metadata and behavioral signals.
Model: claude-opus-4-6
3
Structured JSON Response
Claude returns a structured JSON verdict including: threat classification, confidence level, reasoning, recommended action (quarantine / monitor / clean), and a plain-language explanation shown in the Threat Detail panel.
Returns: JSON verdict

Step 3 — Always-on Protection

Real-time File Watcher & Ransomware Guard

Sentinel monitors your file system in real time using an efficient event-driven watcher — catching threats the moment they appear, not after the damage is done.

500ms Debounce
File events are debounced over 500ms to prevent redundant scans during rapid writes, keeping CPU usage minimal during large copy operations.
Ransomware Guard
Process event monitoring detects ransomware-style behavior — more than 50 file modification events within 10 seconds in the same directory triggers an immediate alert and protection lockdown.
Configurable Watch Paths
Add and remove directories from the Settings panel. The watcher activates instantly on new paths, with no restart required.
Activity Feed
Every scan event, detection, and system action is logged in the live activity feed with file path, severity, and timestamp — searchable and filterable by type.
Desktop Notifications
Critical and high-severity detections surface as native OS notifications so you're alerted immediately — even when the Sentinel window is minimized.
System Tray
Sentinel runs quietly in the system tray. Protection status, quick-scan shortcut, and toggle controls are one click away without opening the full UI.

Step 4 — Threat Containment

AES-256 Quarantine

Quarantine doesn't delete — it isolates. Files are encrypted and locked away so you can investigate, restore, or permanently remove them on your terms.

1
File Isolation
When a file is quarantined, Sentinel immediately moves it out of its original location, preventing execution. The original path and all metadata are preserved in the database.
File cannot execute in quarantine
2
AES-256-CBC Encryption
The file is encrypted using AES-256-CBC. The encryption key is derived via PBKDF2 from a fixed seed, ensuring the file is unreadable outside of Sentinel's quarantine manager.
AES-256-CBC + PBKDF2 key derivation
3
Restore or Delete
From the Quarantine panel you can review each file's threat report, restore it to its original location if it was a false positive, or permanently delete it. Bulk actions supported.
Full control — restore or delete

Step 6 — Getting Started

Setup & Configuration

Sentinel is ready to protect with zero configuration. API keys are optional and unlock cloud-assisted detection layers.

1
Install & Launch
Download the installer for your platform (Windows, macOS, or Linux). Run the installer and Sentinel starts protecting immediately — no configuration required. The signature database is bundled and ready.
Zero config required
2
Add Watch Paths (Optional)
Head to Settings → Watch Paths to add the directories you want monitored in real time. By default, Sentinel watches your Desktop and Downloads folders.
Settings → Watch Paths
3
Add API Keys (Optional)
To unlock VirusTotal multi-engine scanning and Claude AI analysis, enter your API keys in Settings → API Keys. Keys are stored securely in your OS keychain — never in plaintext.
Settings → API Keys
4
Keep Signatures Fresh
Run npm run update-signatures periodically to pull the latest threat intelligence from MalwareBazaar into your local signature database.
Recommended: weekly updates
