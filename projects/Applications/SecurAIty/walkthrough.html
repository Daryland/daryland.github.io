Alert Feed

The Alerts tab shows every detection event in reverse chronological order. CRITICAL alerts pulse red to draw immediate attention.

Severity Levels
CRITICALscore ≥ 85% — auto-blocked
HIGHscore 65–84% — alert + notify
MEDIUMscore 40–64% — watch/log
LOWscore < 40% — informational
a
Search and filter using the toolbar. Filter by ALL, CRITICAL, HIGH, MEDIUM, or LOW. The search box matches any field — try searching a process name, path, or IP address.
b
Click any alert to open the Detail panel on the right. It shows the full context: PID, process path, YARA rules matched, IOC hits, network destination, and the action taken.
c
Actions in the detail panel — Quarantine File moves the suspicious file to the encrypted vault; Kill Process immediately terminates the flagged process (requires confirmation).

AI Alert Explanations

Requires Ollama

Every alert has an Explain with AI button that sends the alert context to a local Ollama model and returns a plain-English explanation — what happened, why it's dangerous, and what to do.

Alert Detail — ProcessHollowing
SeverityCRITICAL
Score94%
Processsvchost.exe
Action TakenQUARANTINE
AI Analysis · Ollama

Process hollowing was detected in svchost.exe, a critical Windows system process. This technique injects malicious code into a legitimate process to hide it from standard process listings, making it appear as trusted Windows infrastructure. This is a hallmark of advanced persistent threats and ransomware loaders. The file has been quarantined — you should run a full system scan and check for persistence mechanisms like scheduled tasks or registry run keys.

Setup Ollama (one time)
1.
Download and install Ollama from ollama.com
2.
Run ollama serve in a terminal to start the local API server
3.
Pull a model: ollama pull llama3.2 (3B, fast) or ollama pull llama3.2:8b (8B, more detailed)
4.
In SecurAIty → Settings → AI Assistant, confirm the model name matches what you pulled
All AI processing happens locally — no data is sent to any cloud service.

Running Scans

The Scan tab lets you run on-demand checks of high-risk system locations. Continuous monitoring runs automatically in the background — scans are for explicit investigation.

Quick Scan

Checks common persistence/threat locations: startup folders, scheduled tasks, temp directories, and known rootkit indicators. Completes in seconds.

Full Scan

Recursively walks the entire filesystem applying ML scoring, YARA rules, and heuristics to every executable. May take several minutes.

Custom Path

Paste any folder path and scan only that directory. Useful for investigating a specific download or suspicious USB drive.

Scan results appear in a list with the file path, threat classification, and score. Any file scoring above 40% has a Quarantine button.

Download Protection

SecurAIty watches your Downloads folder in real time. When an executable file lands on disk, a full-screen alert appears before you can run it.

suspicious_tool.exe
2.4 MB · .exe · Windows Executable
Quarantine

Moves the file to the encrypted vault. You can review and restore it later if it turns out to be legitimate.

Delete

Permanently removes the file from disk. Use when you are certain it is malicious.

Allow

Dismisses the alert and lets the file stay. Use for legitimate software downloads you trust.

Monitored extensions: .exe .msi .dll .bat .cmd .ps1 .vbs .jse .hta .sh .pkg .dmg .deb .rpm .AppImage. On Windows, .js is also monitored (executable by Windows Script Host).

How Detection Works

SecurAIty's threat score is an additive model combining multiple independent detection layers. A threat must score above the configured threshold on the combined score, not any single layer.

50%
max weight
ML Classifier (ONNX / LightGBM)

Trained on the EMBER dataset, a corpus of 1M+ labeled Windows PE files. Extracts 2,381 features from the binary's headers, imports, exports, section entropy, and byte histograms. Outputs a 0–0.5 maliciousness probability.

30%
max weight
YARA Rule Matching

Includes rules for Pegasus spyware, ransomware families, rootkit signatures, and backdoor patterns. Each matching rule adds 0.10 to the score (capped at 0.30). Custom rules can be added in Enterprise tier.

25%
max weight
Behavioral Heuristics

Pattern matching against known-malicious behaviors: process hollowing indicators, DLL injection markers, persistence installation, credential access patterns, network beaconing intervals, and privilege escalation sequences.

40%
max weight
IOC Database Matching

Matches IPs, domains, and file hashes against a curated database of known-malicious indicators of compromise. Each IOC hit adds 0.20 to the score (capped at 0.40). Database updated in Pro/Enterprise tiers.

Decision Thresholds
≥ 85%→AUTO_BLOCK / QUARANTINE / ROLLBACK
≥ 65%→ALERT + notify (configurable)
≥ 40%→WATCH (logged only)
< 40%→IGNORE

Frequently Asked Questions

Does SecurAIty conflict with Windows Defender / other AV?
SecurAIty is complementary to traditional AV, not a replacement. It focuses on behavioral detection and ML scoring rather than signature databases. Most users run both. If you see performance issues, add securaity-agent to your AV's exclusion list.
How do I update the YARA rules and ML model?
In the Free tier, updates come as new application releases. Download the latest version from the GitHub releases page. Pro/Enterprise tiers receive out-of-band rule and model updates automatically.
Why does it show a demo mode / fake alerts?
If the background agent service cannot be reached, the UI falls back to a demo mode that generates simulated alerts so you can explore the interface. Ensure the agent is installed and running as a service. Check the Dashboard for the agent status indicator.
I got a false positive on a legitimate program. What do I do?
Go to Settings → Process Allowlist and add the process name. For quarantined files, open the Quarantine Vault in the Scan tab and click Restore. Please also open a GitHub issue so we can tune the detection — false positive reports directly improve the model.
How do I completely uninstall SecurAIty?
Use the system installer/uninstaller. On Windows: Settings → Apps → SecurAIty → Uninstall. Then manually delete %PROGRAMDATA%\SecurAIty\ to remove all event data. On Linux: sudo dpkg -r securaity then sudo rm -rf /var/lib/securaity /etc/securaity.
