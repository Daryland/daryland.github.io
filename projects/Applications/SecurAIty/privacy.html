Privacy Policy

SecurAIty EDR is designed from the ground up with privacy as a first principle. Most telemetry and data collection found in commercial security tools does not exist here. This document explains exactly what the software does — and does not — do with your data.

Data Collected Locally

SecurAIty collects and stores the following data on your device only, in an AES-256-GCM encrypted SQLite database. This data is never transmitted to any external server in the Free tier.

Process Events

Process name, PID, path, parent process, command-line arguments, user/session context, creation timestamps. Collected to detect malicious process behavior, hollowing, injection, and persistence techniques.

File System Events

File paths accessed, created, modified, or deleted by monitored processes. SHA-256 hashes of suspicious files. File contents are not stored unless a file is moved to the quarantine vault by your explicit action.

Network Events

IP addresses, ports, hostnames, and protocol types for outbound connections flagged as suspicious. Packet contents are not captured or stored.

Registry Events (Windows)

Registry key paths and value names for keys associated with persistence, credential theft, or known malware techniques. Registry values are not stored unless directly relevant to a detected threat.

Threat Scores & Actions

ML model output score (0.0–1.0), contributing factors (YARA matches, heuristic flags, IOC hits), and the automated action taken (BLOCK, QUARANTINE, ALERT, WATCH). Stored for your review in the Alerts tab.

What We Don't Collect

The following data is never collected, stored, or transmitted by SecurAIty:

✗ Usernames or passwords
✗ Browser history or cookies
✗ Email or message content
✗ Clipboard contents
✗ Keystrokes or input events
✗ Screenshots or screen recordings
✗ Document or file content (outside quarantine)
✗ Location or GPS data
✗ Camera or microphone access
✗ Hardware identifiers (MAC, serial)
✗ Usage analytics or feature telemetry
✗ Crash reports sent externally

Network Activity

SecurAIty makes the following outbound network requests. All requests are initiated only by explicit user action unless otherwise noted.

Destination When Data Sent User-Triggered

github.com (releases API) Update check Current version string Yes — on launch

No background telemetry, heartbeat pings, or license verification calls are made. The update check sends only your current version number to compare against the GitHub releases API — no device identifier is included.

AI Features (Ollama)

The "Explain with AI" feature uses Ollama, a local large-language model runtime that runs entirely on your machine. No data is sent to any cloud AI provider (OpenAI, Anthropic, Google, etc.).

What is sent to Ollama: Alert metadata only — event type, severity, threat score, description, process name, file path, network destination, YARA rule names, and the action taken. File contents, packet data, and credentials are never included.

Ollama runs on localhost:11434 by default. You can configure a different endpoint in Settings → AI Assistant if you run Ollama on a different host, but be aware that using a remote Ollama endpoint will send alert data to that host.

Download Protection

SecurAIty watches your Downloads folder for executable file types. When a risky file is detected:

  • A prompt is shown asking you to Quarantine, Delete, or Allow the file
  • The file path, name, extension, and size are temporarily held in memory to power the prompt
  • If you choose Quarantine, the file is moved locally to the quarantine vault — it is not scanned by any cloud service
  • If you choose Allow, the file is left untouched and no record is written

No file content is uploaded during this process. The watcher only reads the file metadata (size) to populate the prompt.

Local Storage & Encryption

All persistent data is stored locally in the following locations:

Windows %PROGRAMDATA%\SecurAIty\events.db  ·  agent.toml  ·  quarantine\
Linux   /var/lib/securaity/events.db  ·  /etc/securaity/agent.toml  ·  quarantine/
macOS   /var/lib/securaity/events.db  ·  /etc/securaity/agent.toml  ·  quarantine/

The events.db SQLite database stores event records encrypted with AES-256-GCM. The encryption key is generated randomly on first run and stored in the agent configuration file. Only the SecurAIty agent process reads or writes this database.

Note: The encryption key is currently stored in the config TOML file. A future update will migrate key storage to the system keychain (Windows DPAPI, macOS Keychain, Linux secret-service) for stronger protection.

Pro & Enterprise Tiers

Optional paid tiers introduce additional features with different privacy implications. These are opt-in only.

Pro $9.99/mo
  • Cloud ML model updates — model weights only, no event data sent
  • VPN killswitch — network interface state only, no traffic inspection
Enterprise $99/user/mo
  • SIEM export — alert metadata forwarded to your configured SIEM endpoint in CEF/JSON/Syslog format
  • REST API — alert data accessible on your local network via authenticated API
  • Custom rules — rule files stored locally, no cloud sync by default

SIEM export destinations are configured by your administrator. Ensure your SIEM policy covers the data fields described in the "Data Collected Locally" section above.

Third-Party Services

URLhaus (abuse.ch)

Used by the Threat Intel tab URL checker. Only the URL string you submit is sent. Governed by the abuse.ch privacy policy. This feature is triggered only by your explicit action — no URLs are submitted automatically.

GitHub Releases API

Used to check for software updates. Only your current version string is included in the request. No device identifiers, OS details, or user data are included. Governed by GitHub's privacy statement.

Ollama (local)

Not a third-party service — Ollama runs entirely on your machine. See ollama.com for Ollama's own privacy terms if you use their hosted models.

Children's Privacy

SecurAIty EDR is a technical security tool intended for adults. It is not directed at children under 13 (or under 16 in the EU/UK) and we do not knowingly collect personal information from minors. If you believe a child has used this software in a way that raises a privacy concern, contact us at the address below.

Your Rights & Data Control

Because all data is stored locally on your device, you have full control at all times:

  • Access: The events database is at the paths listed above. You can read it with any SQLite browser.
  • Delete: Remove events.db to wipe all event history. Remove the quarantine folder to delete quarantined files.
  • Export: Use sqlite3 events.db .dump > export.sql to export all stored events.
  • Uninstall: Uninstalling the application and deleting the data directories above removes all data completely.
